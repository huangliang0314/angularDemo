1.1 nodejs简介

脚本语言需要一个解析器才能运行，JavaScript是脚本语言，在不同的位置有不一样的解析器，如写入html的js语言，浏览器是它的解析器角色。 而对于需要独立运行的JS，nodejs就是一个解析器。

每一种解析器都是一个运行环境，不但允许js定义各种数据结构，进行各种计算， 还允许js使用允许环境提供的内置对象和方法做一些事情。如运行在浏览器中的js的用途是操作DOM，浏览器就提供了document之类的内置对象。 而运行在nodejs中的js的用途是操作磁盘文件或搭建http服务器，nodejs就相应提供了fs,http等内置对象。

并且node有两大特点：
　　 1、基于事件驱动；
　　 2、无阻塞。
 从而nodeJS非常适合处理并发请求,可以实现高性能Web服务器。

1.2 npm简介

NPM（node package manager），通常称为node包管理器。等同于visual studio中的Nuget

			npm的安装、卸载、升级、配置

1.2 node/npm安装

nodejs官网下载对应安装版本，安装即可

1.2.2 npm包的安装模式：本地 vs 全局

    　　　　
  • 本地安装：package会被下载到当前所在目录，也只能在当前目录下使用。
    命令： npm install pkg
    • 　　　　
  • 全局安装：package会被下载到到特定的系统目录下，安装的package能够在所有目录下使用。
    命令： npm install -g pkg
    • 　　
什么时候用本地／全局安装？
    　　　　
  • 当你试图安装命令行工具的时候，例如 angular/cli的时候，使用全局安装
    • 　　　　
  • 当你试图通过npm安装某个模块时，使用本地安装
    • 　　

1.2.3 npm常用命令

npm常用命令
命令作用备注
npm install [-g] [-save-dev] 安装第三方库包
npm install @angular/cli
本地安装最新版本的angular/cli
npm install -g @angular/cli
全局安装最新版本的angular/cli
npm install @angular/cli@""
安装指定版本的angular/cli
npm install @angular/cli --save
将保存配置信息到package.json（package.json是nodejs的配置文件）.
npm install @angular/cli --save-dev
保存至package.json的devDependencies节点，不指定-dev将保存至dependencies节点；
npm update [-g] [-save-dev] 更新指定库包
npm install 通过package.json进行安装 如果我们的项目依赖了很多package，一个一个地安装那将是个体力活。
我们可以将项目依赖的包都在package.json这个文件里声明，然后一行命令搞定
npm uninstall [-g] [-save-dev] 卸载指定第三方库包
npm root [-g] npm根目录；-g是全局根目录
npm install --help 列出所有 npm install 可能的参数形式
